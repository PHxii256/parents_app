AUTH: mobile app vs Postman (integration suggestions)
================================================================================
Source of truth for implemented auth contracts: backend_endpoints.txt §A
(lines up with SafeRoute_auth.postman_collection.json; §A status table marks guardian A5 NEEDS WORK).

Current mobile (AuthDataProvider):
  — baseUrl: http://localhost:5000  (no /api/v1)
  — POST /login  →  should be POST /api/v1/{driver|guardian}/auth/login (pick path by app build
     or login screen selector if one binary serves all roles)
  — POST /login with { accessToken, refreshToken }  →  NOT in Postman; confirm refresh endpoint
     with backend or use silent refresh pattern they provide.
  — POST /logout with body tokens  →  Postman does not show logout; confirm path + method.
  — GET /otp with body  →  Postman uses POST login for OTP trigger and POST verify; align app
     OTP screens with: login → OTP sent OR verify → set-initial-password.
  — POST /reset-password with otp  →  Backend uses verify + set-initial-password + change-password;
     map "forgot password" UX to the real flow.

What to change (high level):
  D — Introduce baseUrl including /api/v1; add AuthRole or flavor enum to build full path:
        /api/v1/driver/auth/login  vs  /api/v1/guardian/auth/login
  D — Parse envelope { success, message, data } and map data.access / data.refresh to JwtStorage
      (not accessToken/refreshToken at top level unless API changes).
  D — Handle dual login response: if data has remaining_time → navigate to OTP; if access+refresh
      → save tokens and fetch user (or decode JWT for user_id + role).
  C — AuthCubit: states for OtpRequired, SetInitialPassword(reset_token), Authenticated.
  P — Wire screens: first-time user vs returning user (match Postman examples).

Keep MOCK logins:
  — Leave AuthRepository mock branches (e.g. e@test.com) returning fake JWTs + User.
  — Flow: try real API first OR use flag useMockAuth=true; on network/5xx fallback optional
    (product decision).

Highlight — files to touch:
  — lib/features/auth/data/provider/auth_data_provider.dart  (paths, envelope, role prefix)
  — lib/features/auth/data/repositories/auth_repository.dart  (parse access/refresh, OTP flow)
  — lib/features/auth/cubit/auth_cubit.dart + auth_state.dart  (new states)
  — lib/core/models/user_data.dart  (may need fields from API /me)
